# Incubator Readiness Checklist

This checklist is intentionally stricter than a minimal application so reviewers can evaluate maturity quickly.

## Repository

- [x] Open-source license.
- [x] README with scope and quickstart.
- [x] Working CLI.
- [x] Tests and 90% coverage target.
- [x] Multi-cloud sample data.
- [x] CI workflow.
- [x] Contribution guide.
- [x] DCO documentation.
- [x] Security policy.
- [x] Governance file.
- [x] Roadmap.
- [x] Threat model.
- [x] Privacy model.
- [x] Release checklist.
- [x] Terraform coverage report and schema.

## OWASP-specific tasks before submission

- [ ] Identify 2-5 project leaders.
- [ ] Confirm leaders are OWASP members if required by current policy.
- [ ] Submit New Project Request through OWASP process.
- [ ] Migrate repository to OWASP GitHub if approved.
- [ ] Create owasp.org project page within required period after access.
- [ ] Replace candidate wording with approved OWASP project branding.
- [ ] Align Code of Conduct with current OWASP Foundation policy.
- [ ] Publish first accepted release notes.
